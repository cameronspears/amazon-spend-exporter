# Security Policy

## Supported versions
- `v0.1.x` is currently supported.

## Reporting a vulnerability
Please do not open public issues for security-sensitive findings.

1. Email maintainers with reproduction details.
2. Include environment, logs, and impact.
3. Allow time for triage and patch before public disclosure.

## Scope notes
This project intentionally avoids credential storage and anti-bot bypass. Reports related to bypassing CAPTCHA/checkpoint protections are out of scope.
